# マーケティング・開発日誌 & 戦略メモ

## 概要
AI活用によるクリエイティブ制作・開発の効率化と、東京美肌堂の各タッチポイント（HP, LP, フォーム, LINE）の改善プロジェクトに関する考察。

## プロジェクト構想
1. **公式LP作成プロジェクト** (現在進行中)
   - 美肌堂の魅力を「正」として伝えるLP。
   - コンテンツをモジュール化し、広告クリエイティブにも転用可能にする。
   - **訴求ポイントの変更**: 「内服が良い」ではなく「**診察がパッとできて10分で終わる**」「**オンライン診察すごい**」という体験の簡便さを強調する。
   - 情報量を絞り、伝えたいことをクリアにする。

2. **予約フォーム改善プロジェクト**
   - 離脱の要因になりやすい箇所。改善のレバレッジが高い。
   - `フォーム1`~`フォーム4`のように複数パターンを作成し、AIにUI/配色の改善案を出させてPDCAを回す。

3. **HP改善プロジェクト**
   - HPを情報のハブとする（最も情報量が多い）。
   - ここから情報を切り出して記事LPなどを作成する運用が理想。

4. **LINE登録導線作成プロジェクト**
   - 直接予約ではなく、LINE登録を挟むフローの検討。
   - オファー内容のAIによる推論と最適化。

## AI/ツール活用方針
- **Cursor**: コーディングだけでなく、企画・ワイヤー作成にも活用。コンテキスト（STP分析、悩みリストなど）を読み込ませて出力させる。
- **Docs**: PDF, Spreadsheet, Web情報をマークダウン化してAIに読み込ませる。
- **Notion AI**: コンテキスト抽出、エージェント作成、チーム共有に強み。

## 現状の課題と解決策
- **課題**: 広告〜LP〜フォームのPDCA効率が悪い。責任の所在が曖昧。
- **解決策**:
    - AIによる高速なLP制作（「2日でLPを作る」）。
    - フォームのUI改善もAI主導で行う。
    - 「アンケートLP」からの脱却。ブランド資産として積み上がるLPへ。

## 具体的なアクション（LP制作において）
- 診察が**10分で終わる**こと、**簡単に手に入る**ことを強調する。
- 既存の「薬の良さ」訴求から「体験の良さ（タイパ）」訴求へシフト。
